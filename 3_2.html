Podłączanie CSS

CSS można dodać do strony internetowej na trzy podstawowe sposoby. Każdy z nich powoduje, że przeglądarka otrzymuje informacje o stylach, ale różnią się one miejscem przechowywania tych stylów, ich skalowalnością oraz zastosowaniem w praktycznych projektach. Zrozumienie tych metod jest bardzo ważne, ponieważ od tego zależy, czy Twoja strona będzie łatwa w rozbudowie, czy stanie się chaotyczna i trudna do utrzymania.

CSS w elemencie (inline CSS)

CSS inline to sposób, w którym style zapisuje się bezpośrednio w znaczniku HTML, używając atrybutu style. Oznacza to, że każdy pojedynczy element ma swoje własne ustawienia wyglądu zapisane dokładnie w miejscu, gdzie ten element się znajduje. Ten sposób jest najbardziej „bezpośredni”, ponieważ przeglądarka nie musi szukać stylów w innych częściach dokumentu — wszystko jest przypisane od razu do konkretnego elementu. Dzięki temu można bardzo szybko zmienić wygląd jednego elementu bez wpływu na inne. Inline CSS jest jednak mało praktyczny w większych projektach, ponieważ jeśli chcemy wystylować wiele elementów w ten sam sposób, musimy powtarzać te same style w wielu miejscach. To prowadzi do chaosu i utrudnia późniejsze zmiany. Wystarczy zmienić jedną rzecz, a trzeba poprawiać wiele elementów osobno. Z tego powodu inline CSS używa się głównie w sytuacjach tymczasowych, testowych albo wtedy, gdy coś ma działać tylko w jednym konkretnym miejscu. Czasem wykorzystuje się go też dynamicznie w JavaScript, gdy styl ma się zmieniać w trakcie działania strony.

CSS wewnętrzny (internal CSS)

CSS wewnętrzny to sposób dodawania stylów bezpośrednio w pliku HTML, w sekcji <head>. Oznacza to, że zarówno struktura strony, jak i jej wygląd znajdują się w jednym dokumencie. Jest to wygodne rozwiązanie na początku nauki CSS, ponieważ wszystko jest w jednym miejscu i łatwo szybko zobaczyć efekty zmian. Można od razu zmieniać kolory, rozmiary czy układ i natychmiast obserwować rezultat. Problem pojawia się wtedy, gdy strona zaczyna się rozrastać. Kod CSS miesza się z HTML, co utrudnia czytanie i porządkowanie projektu. Dodatkowo nie da się łatwo używać tych samych stylów na wielu stronach bez kopiowania kodu. Z tego powodu internal CSS stosuje się głównie w małych projektach lub do nauki.

CSS zewnętrzny (external CSS)

CSS zewnętrzny polega na trzymaniu stylów w osobnym pliku .css, który jest podłączany do HTML. Dzięki temu HTML odpowiada tylko za strukturę, a CSS tylko za wygląd. To rozwiązanie jest standardem w profesjonalnych projektach, ponieważ pozwala utrzymać porządek i łatwo zarządzać stylem całej strony. Jeden plik CSS może obsługiwać wiele stron, więc zmiana jednego stylu wpływa na cały projekt. Największą zaletą tego podejścia jest skalowalność — im większy projekt, tym bardziej widać jego sens. Kod jest czystszy, łatwiejszy do utrzymania i wygodniejszy w pracy zespołowej. Minusem jest tylko to, że trzeba nauczyć się organizacji plików i ich łączenia, ale to szybko staje się naturalne.

Przykłady użycia danych podłączeń CSS:

CSS w elemencie (inline CSS)

<p style="color: red;">To jest czerwony tekst</p>

To jest czerwony tekst

CSS wewnętrzny (internal CSS)

<!DOCTYPE html>
<html>
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>

<body>
  <p>To jest zielony tekst</p>
</body>
</html>

To jest zielony tekst

CSS zewnętrzny (external CSS)

Kod w HTML:

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="style.css">
</head>

<body>
  <p>To jest żółty tekst</p>
</body>
</html>

Kod w arkuszu stylów style.css

p {
    color: yellow;    
}

To jest żółty tekst
