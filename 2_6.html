Formularze

Formularze w HTML są jednym z najważniejszych elementów stron internetowych. Dzięki nim użytkownik może wprowadzać dane, które następnie mogą zostać wysłane do serwera lub przetworzone przez stronę. W HTML formularz tworzy się za pomocą znacznika <form>. Wewnątrz niego umieszcza się różne pola, takie jak pola tekstowe, przyciski, listy wyboru czy checkboxy. Użytkownik wpisuje dane w formularzu, a następnie zazwyczaj wysyła je przyciskiem „Wyślij”. Dane te mogą trafić do serwera (np. w przypadku logowania) lub zostać obsłużone przez JavaScript.

<form>

Znacznik <form> to podstawowy element każdego formularza w HTML. Otacza on wszystkie pola, przyciski i inne elementy, które mają służyć do zbierania danych od użytkownika. Można powiedzieć, że <form> działa jak kontener, który zbiera dane z pól i decyduje, co się z nimi stanie po wysłaniu. Bez <form> pola takie jak inputy czy checkboxy nie miałyby „logicznego sensu”, bo nie byłoby sposobu ich wspólnego wysłania.


Najważniejsze atrybuty <form>:

  • action, które określa, gdzie mają trafić dane po wysłaniu formularza
  • method, które określa sposób wysyłania danych, ma dwie najważniejsze wartości:
    • get, które powoduje, że dane trafiają do URL i są widoczne w pasku adresu
    • post, które powoduje, że dane są przekazywane w treści żądania HTTP i nie są widoczne w URL

<input>

Znacznik <input> to jeden z najważniejszych elementów formularzy HTML. Służy do tworzenia pól, w których użytkownik może wpisywać dane lub wybierać różne opcje. To właśnie dzięki <input> użytkownik może podać np. swoje imię, hasło, numer telefonu albo zaznaczyć checkbox.

Najważniejsze atrybuty <input>:

  • type określa rodzaj pola:
    • button to zwykły przycisk (często używany z JavaScript do akcji)
    • checkbox pole wyboru (można zaznaczyć kilka opcji)
    • color wybór koloru (paleta kolorów)
    • date wybór daty (dzień, miesiąc, rok bez godziny)
    • datetime-local wybór daty i godziny (bez strefy czasowej)
    • email pole do wpisania adresu e-mail
    • file wybór pliku z komputera (upload)
    • hidden ukryte pole (niewidoczne dla użytkownika)
    • image obrazek działający jako przycisk wysyłania
    • month wybór miesiąca i roku
    • number pole do wpisywania liczb
    • password pole hasła (tekst ukryty)
    • radio przycisk wyboru jednej opcji
    • range suwak do wybierania wartości
    • reset przycisk resetujący formularz
    • search pole do wyszukiwania
    • submit przycisk wysyłający formularz
    • tel pole na numer telefonu
    • text standardowe pole tekstowe (domyślne)
    • time wybór godziny
    • url pole na adres strony internetowej
    • week wybór tygodnia i roku
  • name nazwa pola, potrzebna do wysyłania danych z formularza
  • placeholder tekst podpowiedzi widoczny w polu
  • required pole obowiązkowe (nie można wysłać formularza bez niego)
  • value domyślna wartość w polu
  • disabled wyłącza pole (nie można go kliknąć ani edytować)
  • readonly pole tylko do odczytu (widać, ale nie można zmienić)
  • min minimalna wartość (dla input typu number lub date)
  • max maksymalna wartość (dla input typu number lub date)
  • maxlength maksymalna liczba znaków w polu tekstowym
  • minlength minimalna liczba znaków wymagana w polu
  • checked domyślnie zaznacza checkbox lub radio
  • list łączy input z listą podpowiedzi (datalist)
  • autocomplete włącza lub wyłącza podpowiadanie przeglądarki
  • autofocus automatycznie ustawia kursor w polu po załadowaniu strony
  • pattern wzór (regex), który musi spełniać wpisany tekst
  • size określa widoczną szerokość pola (ilość znaków)
  • step określa krok zmiany wartości (np. co 2 w number)
  • width szerokość (działa głównie przy input typu image)
  • multiple pozwala wybrać wiele plików lub wartości (np. w file, email)

<textarea>

<textarea> to element formularza używany wtedy, gdy użytkownik ma wpisać dłuższy tekst, np. wiadomość, komentarz albo opis. W przeciwieństwie do <input type="text">, który jest jednowierszowy, <textarea> pozwala pisać w wielu liniach.

Najważniejsze atrybuty <textarea>:

  • cols, czyli liczba kolumn
  • rows, czyli liczba wierszy
  • name, czyli nazwa pola tekstowego

<button>

<button> to element HTML używany do tworzenia przycisków, które mogą wykonywać różne akcje: wysyłać formularz, resetować dane albo uruchamiać JavaScript. W przeciwieństwie do <input type="submit">, <button> jest bardziej elastyczny, bo może zawierać tekst, ikony, a nawet inne elementy HTML.

Najważniejsze atrybuty <button>:

  • type="submit" wysyła formularz
  • type="reset" - resetuje formularz
  • type="button" - zwykły przycisk (bez działania)

<select>

<select> to element formularza służący do tworzenia list rozwijanych. Dzięki niemu użytkownik może wybrać jedną lub kilka opcji z gotowej listy. Jest bardzo często używany w formularzach, ponieważ pozwala zaoszczędzić miejsce i uporządkować wybory. W środku <select> wpisujemy <option>, aby utworzyć poszczególne pozycje listy rozwijanej.

Najważniejsze atrybuty <select>:

  • name, czyli nazwa pola tekstowego
  • multiple, które pozwala wybrać kilka opcji jednocześnie
  • size, które określa ile opcji ma być widoczne jednocześnie
  • required, czyli musi zostać wybrana jakaś opcja

Najważniejsze atrybuty <option>:

  • value, czyli wartość, która jest wysłana przez formularz
  • selected, czyli opcja jest domyślnie wybrana
  • disabled, czyli opcja nie może zostać wybrana

<optgroup> to element używany wewnątrz <select>, który pozwala grupować opcje w listach rozwijanych. Dzięki temu lista staje się bardziej uporządkowana i czytelna dla użytkownika.

Najważniejszym atrybutem <optgroup> jest label, który określa nazwę grupy widoczną na liście.

<fieldset>

<fieldset> to element HTML używany do grupowania powiązanych elementów formularza. Dzięki niemu formularz staje się bardziej uporządkowany i czytelny. Przeglądarka automatycznie dodaje wokół niego ramkę, która wizualnie oddziela daną sekcję formularza. Wewnątrz <fieldset> używa się znacznika <legend>, który służy do nadawania nazwy sekcji formularza.

<label>

<label> to element HTML używany do tworzenia etykiet dla pól formularza. Dzięki niemu użytkownik wie, do czego służy dane pole. <label> służy głównie do opisywania pól typu input, checkbox, radio, textarea, select. Dzięki <label> kliknięcie w tekst zaznacza pole i strona jest bardziej dostępna.

<output>

<output> to element HTML używany do wyświetlania wyniku działania formularza lub skryptu. Najczęściej używa się go razem z JavaScript, aby pokazywać użytkownikowi wynik obliczeń lub aktualną wartość.

Najważniejsze atrybuty <output>:

  • name, czyli nazwa wyniku używana np. w JavaScript
  • for, które określa, z jakimi elementami output jest powiązany
  • form, które łączy output z konkretnym formularzem
